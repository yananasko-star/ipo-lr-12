# Задание 2: Базовый класс Vehicle

## Описание задания
**Создать базовый класс Vehicle для транспортного средства.**

### Требования к классу:

#### Атрибуты:
- vehicle_id – уникальный идентификатор (строка, генерируется случайно при создании)
- capacity – грузоподъемность (в тоннах)
- current_load – текущая загрузка (по умолчанию 0)
- clients_list – список клиентов чьи грузы загружены

#### Методы:
- load_cargo(client) – увеличивает загрузку на cargo_weight из объекта client и добавляет клиента в список клиентов (с проверкой на не превышение грузоподъемности и валидацией типов данных)

#### Магический метод:
- __str__(): возвращает строку с ID транспорта, его грузоподъемностью и текущей загрузкой

### Требования:
1. Все методы должны иметь валидацию данных
2. ID генерируется случайно при создании объекта
3. Метод load_cargo() должен проверять:
   - Не превышает ли новая загрузка грузоподъемность
   - Корректность типа объекта client (должен быть экземпляром Client)

### Пример использования:
`python
from task_1.transport.client import Client
from task_2.transport.vehicle import Vehicle

# Создание клиента
client = Client("Иван Иванов", 5.0)

# Создание транспортного средства
vehicle = Vehicle(20.0)  # Грузоподъемность 20 тонн
print(vehicle)
# Вывод: Транспорт VH-A1B2C3D4: грузоподъемность 20.0т, текущая загрузка 0.0т

# Загрузка груза
vehicle.load_cargo(client)
print(vehicle)
# Вывод: Транспорт VH-A1B2C3D4: грузоподъемность 20.0т, текущая загрузка 5.0т

# Проверка валидации
try:
    vehicle.load_cargo("не клиент")  # Ошибка: неверный тип
except TypeError as e:
    print(f"Ошибка: {e}")

try:
    vehicle2 = Vehicle(5.0)
    client2 = Client("Петр", 10.0)
    vehicle2.load_cargo(client2)  # Ошибка: перегрузка
except ValueError as e:
    print(f"Ошибка: {e}")
